# Rubric Validation Revision Log

This file records every revision made to rubric YAML definitions or judge prompts
during the validation process. It is a required paper appendix artifact.

If no revisions were needed (all 4 gates passed on first annotation round), this
file remains empty below the header.

## Format

Each entry should include:
- **Date**
- **Gate that failed** (R1 / R2 / R3 / R4)
- **What changed** (rubric YAML diff, prompt diff, plain-language guide update)
- **Metric before and after** (ICC/α/std/τ_b before revision → after)
